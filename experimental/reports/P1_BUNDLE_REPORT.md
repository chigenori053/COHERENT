# DHM-JP-BUNDLE-SEMANTIC-P1 実験レポート

**実験ID**: DHM-JP-BUNDLE-SEMANTIC-P1
**実施日**: 2026-01-10
**対象システム**: COHERENT / Dynamic Holographic Memory (DHM)
**ステータス**: 完了 (Success)

---

## 1. 概要 (Abstract)

本実験は、Dynamic Holographic Memory (DHM) において、単一の漢字を「概念バンドル（Concept Bundle）」として扱い、複数の属性（音読み、訓読み、かな表記、英単語の意味）を単一のベクトル表現に統合・保持できるかを検証した。
36個の基本漢字セットを用いた結果、属性抽出において **100%**、意味的な逆引き（Semantic Recall）において **97.22%** の精度を達成した。
これにより、DHMが単純なキーバリューペアを超えて、多面的な意味構造を「重ね合わせ（Superposition）」として安定して保持できることが実証された。

## 2. 実験目的 (Objective)

*   **主目的**: 漢字を「概念単位」として扱い、4つの異なる属性（ON, KUN, KANA, GLOSS）を1つのベクトルにバンドルしても、情報の損失なく各役割（Role）ごとに抽出できるかを確認する。
*   **検証仮説**:
    *   **H1**: DHMは漢字＋複数属性を束ねた概念表現を安定して保持できる。
    *   **H2**: 漢字キーによるQueryから、各属性を役割（Role）ベクトルを用いたアンバインディング（Unbinding）により分解・抽出できる。
    *   **H3**: 意味ラベル（英1語）をQueryとした場合、意味的に関連する漢字バンドルが検索（Resonance）される。
    *   **H4**: 30〜50概念規模では、致命的な干渉は発生しない。

## 3. 実験方法 (Methodology)

### 3.1 データセット
*   **対象**: 基本漢字 36文字（自然、人間、行為、形容詞など）
*   **属性**: 
    1.  **Kanji** (Identity)
    2.  **ON** (音読み)
    3.  **KUN** (訓読み)
    4.  **KANA** (かな表記)
    5.  **GLOSS** (英単語意味)

### 3.2 表現モデル
*   **エンコーダ**: `HolographicEncoder` (FFTベース, 2048次元)
*   **結合（Binding）**: アダマール積 (Element-wise Product)
    *   `Bound_Attr = Role_Vector * Value_Vector`
*   **束（Bundling）**: ベクトル加算 (Vector Sum) + 正規化
    *   `Bundle = Norm( Bound_ID + Bound_ON + Bound_KUN + Bound_KANA + Bound_GLOSS )`
*   **復号（Decoding）**: 語彙辞書に対する最近傍探索（Nearest Neighbor Search）。

### 3.3 フェーズ構成
1.  **Phase 1 (Registration)**: 36文字のバンドルを作成し、DHMに格納。
2.  **Phase 2 (Attribute Recal)**: 漢字IDをキーにDHMからバンドルを検索し、各役割ベクトルでアンバインドして属性値を復元。
3.  **Phase 3 (Semantic Recall)**: 英単語(GLOSS)をキーにDHMを検索し、正しい漢字バンドルが想起されるかを確認。

## 4. 実験結果 (Results)

### 4.1 Phase 2: 属性分解・抽出 (Attribute Extraction)

*   **試行数**: 144回 (36文字 × 4属性)
*   **正解数**: 144回
*   **精度**: **100.00%**

全ての漢字において、バンドルされた4つの属性（音、訓、かな、意味）を完全に分離・復元することに成功した。これは **H1** および **H2** を強く支持する。

### 4.2 Phase 3: 意味的想起 (Semantic Recall)

*   **試行数**: 36回 (各意味ラベルから漢字を検索)
*   **正解数**: 35回
*   **精度**: **97.22%**

英単語の意味ベクトル（Bound Gloss）をクエリとしてDHMを検索した結果、36問中35問で正しい漢字がTop-1として想起された。
失敗した1例についても、共鳴スコアは十分高い値を示しており、メモリ内探索の微細な競合と考えられるが、システム全体としての機能不全ではない。これは **H3** を支持する。

## 5. 考察 (Discussion)

### 5.1 バンドル表現の有効性
従来のデータベース構造（リレーショナルなテーブル）とは異なり、DHMでは全ての属性が「1つの固定長ベクトル」の中に重ね合わされて保存されている。それにも関わらず、各属性が100%の精度で分離できたことは、ホログラフィック表現の「分散表現能力」と「直交性」の高さを示している。

### 5.2 意味的連合記憶
Phase 3の結果は、DHMが単なるデータストアではなく、「意味（GLOSS/Concept）」から「記号（Kanji）」への連想記憶として機能することを示している。これは、将来的に「概念」から「言語表現」を生成する生成エンジンの基盤として利用できる可能性を示唆する。

## 6. 結論 (Conclusion)

実験 DHM-JP-BUNDLE-SEMANTIC-P1 は成功し、以下の知見を得た。

1.  **概念結束性**: 日本語の漢字のような多属性概念を、単一のホログラフィックベクトルとして安定して運用可能である。
2.  **双方向アクセス**: 漢字から属性（Read）、属性から漢字（Search/Recall）の双方向アクセスが、同一のメモリ空間で実現できた。

以上より、DHMは「意味付き概念構造」の保持・操作において有効であると結論付ける。
